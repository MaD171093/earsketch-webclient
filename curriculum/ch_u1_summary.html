Unit 1 Summary

Let’s review the material covered so far.

Chapter 1:

  • Initially, we covered the various modules of the EarSketch workspace. 1.3

  • We learned about the digital audio workstation (DAW) and how to run a script to make music. 1.4, 1.6

  • Programming is the process of designing, writing, testing, debugging, and maintaining code within a computer program. 1.7

  • The code editor is where an EarSketch script is written. Comments, setup, music, and finish are the basic sections of a script. 1.8

  • The fitMedia() function provides the basis for composing in EarSketch. 1.9

Chapter 2:

  • Rhythm is the arrangement of sound as music flows through time. A beat is the primary unit of time. 2.1

  • Data types are the fundamental way of representing information in a program. 2.2

  • Functions are the primary way of supplying information to the computer. 2.3

  • Variables are used to store information in memory. 2.5

Chapter 3:

  • Debugging is the process of finding and fixing errors. It is used throughout the programming world, not just in EarSketch. We learned about syntax errors, runtime errors, and logic errors. 3.1

  • The console supplies information about the state of program, making it a useful debugging tool. 3.2

  • Comments are inserted to make code easier to understand. They keep code organized and readable. 3.3

Chapter 4 and Chapter 5:

  • setEffect() can be used to modify sounds in numerous ways. Supplying different arguments to setEffect() changes the effects it produces. 4.1

  • setEffect() has an extended set of arguments that are used to create envelopes, enabling effects that change over time. 5.2

Chapter 6:

  • The EarSketch sound library is organized by genre. 6.1

  • Sounds within a particular folder are in the same key, or group of pitches. This means that they will sound good when used together. 6.2

  • Transitions create a natural flow between sections of a piece. We experimented with a number of transition strategies. 6.3

Chapter 7:

  • You have ownership over the music you create in EarSketch. Copyright protects your work, defining how your music can be used by others. 7.1

  • It is important to obtain permission before using another person’s work. Licensing allows you to easily define permissions. 7.6

  • Willingness to share music contributes to a more free, creative environment.

Chapter 8:

  • It is important to take a step back and evaluate your work. This ensures that your code is correct and complete. These two concepts define the quality of a script. 8.1
